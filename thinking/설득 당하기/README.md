## 설득 당하기 

```
저는 신입과 1년차의 경력의 차이점중 하나가 개발 실력도 있겠지만 제품과 서비스를 만들어가면서 누군가와 같이 문제를 해결하는 능력,
조직과 팀내에 협업하는 능력을 키울 수 있고 대표적으로 “누군가를 설득하는 능력“을 키울 수 있다고 생각을 하는데요.

아래의 글은 누군가를 설득하는 것도 중요하지만 누군가에게 설득되는 것도 중요하다는 것을 보여주는 글입니다.

“설득 되기” 라는 말이 조금 아이러니하게 들릴수 있겠지만
협업을 잘하는 사람은 누군가를 잘 설득하는 사람이 아니라 문제에 대해 옳고 그름을 잘 판단하고 제안하거나 선택할수 있는 사람이죠.

“설득 되기“를 통해 개인의 성장을 도모할 수 있고 좋은 조직을 만들어가고 싶다면 팀원의 동기부여를 갉아먹는 무조건적인 비난을 피하고
“내가 어떤 의견을 말해도 괜찮다“라는 환경을 만들어줄수 있습니다.
당연히 무조건적인 수용은 피해야 하지만
때로는 문제를 같이 해결함으로써 더 나은 조직을 위해 “설득 되기“의 능력을 연습해보면 어느 순간
시니어라면 존경을, 주니어라면 인정을 받고 리더라면 신뢰를, 그리고 팀원이라면 심리적 안전감을 얻을수 있지 않을까요?
```
